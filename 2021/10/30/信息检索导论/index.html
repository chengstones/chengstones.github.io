Chapter 1 布尔检索

  1. 信息检索（IR）：

    • 大规模非结构化数据
    • 满足用户需求
    • 特定资料

  2. 数据类型

    • 结构化 - 关系数据库
    • 非结构化
    • 半结构化 - 网页界面

  3. 分类

    • 给定类别体系（标注）
    • 扩展 （类似于 ML 领域的分类概念）

  4. 线性扫描（×）

    • 数据量
    • 匹配方式灵活
    • 打分排序

  5. 词项-文档关联矩阵：布尔值

  6. 布尔值查询 逻辑连接词

  7. 统计信息

    • 正确率：返回的结果中真正和信息需求相关的文档所占的百分比

    • 召回率：所有和信息需求真正相关的文档中被检索系统返回的百分比

  8. 倒排索引

    • 关联矩阵的稀疏性 -> 倒排表

    • 倒排记录：词项 -> 文档编号

    • 构建倒排索引：收集文档 -> 词条化 -> 语言学处理 -> 倒排索引

    • 索引 — 倒排记录表 && 词项词典

  9. 布尔查询：对倒排记录表进行合并，例如：AND -> 取交

  10. 倒排记录表

    • 单链表 -> 文档的插入和更新
    • 变长数组 -> 节省指针内存消耗，访问速度快
    • -> 定长数组的链表

  11. 合并 -> 基于文档频率的启发式算法

  12. 基于基本布尔操作的扩展

    • 排序检索模型

  13. Future Key_Points

    1. 容忍错误 模糊查询
    2. 改善结构 支持复合查询
    3. 非二值 引入词项频率
    4. 评分

Chapter 4 索引构建

  1. 硬件基础：
    1. 访问内存 快于 访问磁盘
    2. 磁盘读写 寻道时间
    3. 操作系统以数据块为单位进行读写
    4. 磁盘 I/O 由系统总线处理 而不是 处理器
    5. IR 系统服务器 内存 >> 系统内存
  2. 基于块排序索引构建算法
    1. 目的：基于磁盘的外部排序算法：减少随机寻道次数
    2. 步骤：分割文档集 -> 排序词项ID - 文档ID 对 -> 将临时排序结果存放到磁盘 -> 合并
  3. 内存式扫描索引构建算法
    1. 目的：消除 词项 -> ID 的数据结构
    2. 原理：每个块采用新字典
  4. 分布式索引构建
    1. 往往按照词项或文档进行分割后分布在多台计算机上
    2. MapReduce架构：面向大规模计算机集群
      1. 将计算任务划分成子任务快
      2. 使得每个工作节点在短时间内能够快速处理
  5. 动态索引构建
    1. 考虑文档集动态变化

Chapter 5 索引压缩

Key word：无损压缩 | 词典和倒排索引 | 高速缓存 | 加快传输 | 统计特性| 词典 -> 长串| 按块存储压缩 |变长字节编码和 γ 编码

  1. 无损压缩 | 有损压缩
  2. Heaps 定律 -> 估计词项数目
    • 文档集大小和词汇量之间在对数空间存在线性关系
    • 两点假设：文档数目 ++ 词汇量++ 且不收敛；大规模文档集 词汇量 极大
  3. Zipf 定律 -> 建模词项分布
    • 如果 t1 是文档集中的出现最多的词项，t2 是文档集中的出现第二多的词项，依此类推，那么，
      排名第 i 多的词项的文档集频率 cfi与 1/i 成正比
  4. 词典压缩
    • 目的：将词典放入内存
    • 方法：
      • 将词典看作单一字符串 -> 定长数组 -> 指针
      • 按块储存 -> 保留第一个词项指针 额外字节保存词项长度 -> 减少词项指针个数
      • 前端编码 -> 公共前缀
      • 最小完美哈希 -> 静态
  5. 倒排记录表压缩
    • 原理：高频词出现的文档 ID 序列值之间相差不大
    • 可变字节码
    • γ 编码
